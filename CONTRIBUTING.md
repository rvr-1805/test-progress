## Commit Message Style

Use simple conventional prefixes:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `refactor:` code restructure
- `chore:` maintenance

## Review Checklist

- Code works locally
- No unrelated file changes
- PR description explains *what* and *why*
- Screenshots added if UI changed
